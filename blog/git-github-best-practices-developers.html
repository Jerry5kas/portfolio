Git & GitHub Best Practices for Developers

AK

Azhagirishankar K

Full Stack Developer

Version control is the backbone of modern software development. Mastering Git and GitHub workflows is essential for any developer working in a team or maintaining professional codebases. In this comprehensive guide, I'll share best practices that have helped me collaborate effectively on 20+ projects and maintain clean, organized repositories.

1. Commit Message Best Practices

Clear, descriptive commit messages are crucial for understanding project history. Follow these conventions:

Conventional Commit Format

Use a consistent format that includes type, scope, and description: 

# Format: <type>(<scope>): <description>

# Examples:
feat(auth): add user authentication with JWT
fix(api): resolve 500 error on user registration
docs(readme): update installation instructions
refactor(database): optimize user queries
test(api): add unit tests for payment module
chore(deps): update Laravel to version 10

Commit Message Types

  • feat: New feature
  • fix: Bug fix
  • docs: Documentation changes
  • style: Code style changes (formatting)
  • refactor: Code refactoring
  • test: Adding or updating tests
  • chore: Maintenance tasks

2. Branching Strategies

A well-defined branching strategy ensures smooth collaboration and code quality:

Git Flow Model

Use a structured branching model for organized development: 

# Main branches
main/master     # Production-ready code
develop         # Integration branch for features

# Supporting branches
feature/*       # New features
bugfix/*        # Bug fixes
hotfix/*        # Urgent production fixes
release/*       # Release preparation

Branch Naming Conventions

# Feature branches
feature/user-authentication
feature/payment-integration

# Bug fix branches
bugfix/login-error
bugfix/memory-leak

# Hotfix branches
hotfix/security-patch
hotfix/critical-bug

3. Pull Request Best Practices

Effective pull requests facilitate code review and collaboration:

PR Title and Description

  • Use clear, descriptive titles
  • Include context and motivation
  • List changes and breaking changes
  • Add screenshots for UI changes
  • Reference related issues

PR Size Guidelines

  • Keep PRs focused and small (200-400 lines ideal)
  • One feature or fix per PR
  • Break large features into smaller PRs

4. Git Workflow Commands

Essential Git commands for daily development:

Starting Work

# Update main branch
git checkout main
git pull origin main

# Create feature branch
git checkout -b feature/new-feature

# Or from develop
git checkout develop
git pull origin develop
git checkout -b feature/new-feature

Making Changes

# Stage specific files
git add path/to/file.php

# Stage all changes
git add .

# Commit with message
git commit -m "feat(auth): add login functionality"

# Push to remote
git push origin feature/new-feature

Keeping Branch Updated

# Fetch latest changes
git fetch origin

# Rebase on main (cleaner history)
git rebase origin/main

# Or merge main into branch
git merge origin/main

5. .gitignore Best Practices

Proper .gitignore files prevent committing sensitive or unnecessary files:

Laravel .gitignore Example

# Dependencies
/vendor
/node_modules

# Environment files
.env
.env.backup
.env.production

# IDE files
.idea
.vscode
*.swp
*.swo

# OS files
.DS_Store
Thumbs.db

# Logs
*.log
storage/logs/*

# Compiled files
/public/hot
/public/storage
/storage/*.key

6. GitHub Collaboration Features

Leverage GitHub features for better team collaboration:

Issues and Project Management

  • Use GitHub Issues for bug tracking
  • Link PRs to issues using keywords (fixes #123)
  • Use labels for categorization
  • Create project boards for task management

Code Review Process

  • Request reviews from team members
  • Address all review comments
  • Use "Request Changes" appropriately
  • Approve only when code is ready

7. Security Best Practices

Protect sensitive information and maintain repository security:

Never Commit Secrets

# ❌ Never commit these:
API keys
Passwords
Database credentials
Private keys
.env files

# ✅ Use environment variables
# Store in .env (gitignored)
# Use GitHub Secrets for CI/CD

Branch Protection Rules

  • Require pull request reviews
  • Require status checks to pass
  • Require branches to be up to date
  • Restrict force pushes

8. Advanced Git Techniques

Power user techniques for complex scenarios:

Interactive Rebase

# Clean up commit history
git rebase -i HEAD~3

# Options:
# pick - use commit
# squash - combine with previous
# edit - modify commit
# drop - remove commit

Stashing Changes

# Save uncommitted changes
git stash

# List stashes
git stash list

# Apply stash
git stash apply

# Apply and remove
git stash pop

Conclusion

Mastering Git and GitHub best practices is essential for professional software development. By following these guidelines, you'll maintain clean repositories, facilitate smooth collaboration, and improve code quality. Remember:

  • Write clear, descriptive commit messages
  • Follow a consistent branching strategy
  • Keep pull requests focused and well-documented
  • Never commit sensitive information
  • Use GitHub features for effective collaboration

These practices have helped me maintain 20+ production projects with clean histories and efficient team workflows. Start implementing these practices today to elevate your version control game!
