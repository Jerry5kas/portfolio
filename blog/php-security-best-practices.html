PHP Security Best Practices: Complete Guide to Secure Coding

AK

Azhagirishankar K

Full Stack Developer

Security is paramount in web development. This comprehensive guide covers essential PHP security practices to protect your applications from common vulnerabilities and attacks.

1. SQL Injection Prevention

Always use prepared statements with PDO or MySQLi:

// ❌ Vulnerable
$query = "SELECT * FROM users WHERE id = " . $_GET['id'];

// ✅ Secure
$stmt = $pdo->prepare("SELECT * FROM users WHERE id = ?");
$stmt->execute([$_GET['id']]);

2. XSS (Cross-Site Scripting) Protection

Escape all user input before displaying:

// Escape output
echo htmlspecialchars($userInput, ENT_QUOTES, 'UTF-8');

// In Laravel
{{ $userInput }} // Auto-escaped
{!! $userInput !!} // Only if trusted

3. CSRF Protection

Use CSRF tokens for all forms:

// Generate token
$token = bin2hex(random_bytes(32));
$_SESSION['csrf_token'] = $token;

// Verify token
if (!hash_equals($_SESSION['csrf_token'], $_POST['csrf_token'])) {
    die('CSRF token mismatch');
}

4. Input Validation

// Validate and sanitize
$email = filter_var($_POST['email'], FILTER_VALIDATE_EMAIL);
$age = filter_var($_POST['age'], FILTER_VALIDATE_INT, [
    'options' => ['min_range' => 1, 'max_range' => 120]
]);

5. Password Security

// Hash passwords
$hash = password_hash($password, PASSWORD_BCRYPT);

// Verify passwords
if (password_verify($password, $hash)) {
    // Login successful
}

6. File Upload Security

  • Validate file types (MIME type, not extension)
  • Limit file size
  • Store uploads outside web root
  • Rename uploaded files
  • Scan for malware

7. Session Security

// Secure session configuration
ini_set('session.cookie_httponly', 1);
ini_set('session.cookie_secure', 1);
ini_set('session.use_strict_mode', 1);
session_regenerate_id(true);

8. Error Handling

Never expose sensitive information in error messages:

// Production
ini_set('display_errors', 0);
error_reporting(0);
log_errors(1);

// Development
ini_set('display_errors', 1);
error_reporting(E_ALL);

9. HTTPS and Secure Headers

// Force HTTPS
if (!isset($_SERVER['HTTPS']) || $_SERVER['HTTPS'] !== 'on') {
    header("Location: https://" . $_SERVER['HTTP_HOST'] . $_SERVER['REQUEST_URI']);
    exit();
}

// Security headers
header('X-Content-Type-Options: nosniff');
header('X-Frame-Options: DENY');
header('X-XSS-Protection: 1; mode=block');

10. Authentication Best Practices

  • Use strong password requirements
  • Implement account lockout after failed attempts
  • Use two-factor authentication
  • Store sessions securely
  • Implement password reset securely

Conclusion

Security should be a priority from the start of development. Following these best practices helps protect your PHP applications from common vulnerabilities and ensures a secure user experience.
