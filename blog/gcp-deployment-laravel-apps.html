Deploying Laravel Apps on Google Cloud Platform: Complete Guide

AK

Azhagirishankar K

Full Stack Developer

Google Cloud Platform offers multiple options for deploying Laravel applications. This guide covers Cloud Run, App Engine, and Compute Engine deployment strategies.

1. Cloud Run Deployment

Serverless container deployment for Laravel:

# Dockerfile
FROM php:8.2-fpm
WORKDIR /var/www/html
COPY . .
RUN composer install --no-dev --optimize-autoloader

# Deploy
gcloud run deploy laravel-app --source .

2. App Engine Deployment

Fully managed platform for Laravel:

# app.yaml
runtime: php82
env: flex

runtime_config:
  document_root: public

env_variables:
  APP_ENV: production
  APP_KEY: your-app-key

3. Compute Engine Setup

Virtual machine deployment with full control:

  • Create VM instance
  • Install PHP, Composer, Nginx
  • Configure SSL certificates
  • Set up database connection
  • Configure firewall rules

4. Cloud SQL Integration

Connect Laravel to Cloud SQL:

DB_CONNECTION=mysql
DB_HOST=/cloudsql/project:region:instance
DB_DATABASE=your_database
DB_USERNAME=your_user
DB_PASSWORD=your_password

5. Environment Configuration

Secure environment variable management:

  • Use Secret Manager for sensitive data
  • Configure .env for production
  • Set up proper caching
  • Configure queue workers

6. Monitoring and Logging

Set up Cloud Monitoring and Logging for your Laravel application.

Conclusion

GCP provides scalable and reliable infrastructure for Laravel applications. Choose the deployment method that best fits your application's needs and scale requirements.
